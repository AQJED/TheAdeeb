Why I stopped coding from scratch

When I started working on simulation programs, I tried to write every part of the data handling myself. I thought that writing my own packing and serialization code would make the program faster or easier to manage. This was a mistake that cost me a lot of time.

The Manual Implementation Trap

I used to manually calculate offsets and bit-shift data for protocols like DIS (Distributed Interactive Simulation). I believed my custom code was better than using a generic library. In reality, I was recreating logic that had already been solved and tested.

Searching the Source Code

Instead of writing a new function to check a message or pack a struct, I now open the library folders. I look through the header files and the source code to see how the original developers solved the problem. Most of the time, the method or function I need already exists inside the library.

Spending thirty minutes reading the library source code is much more productive than writing new code from zero. It helps me understand the logic and prevents me from creating bugs in the serialization process. 

// Example: Checking the PDU type using existing library methods
if (pdu->getPduType() == PduType_EntityState) {
    // Logic is already handled by the library classes
}

Reliability of Established Code

Libraries like CCL or Open-DIS are written by large teams with a lot of experience. They have tested their code in many different scenarios and handled edge cases that I might not notice until a simulation fails during a demo. These libraries are stable and often include validation checks that are difficult to write manually.

When to Customize

There are rare cases where a library does not fit the specific needs of a project perfectly. In these situations, I modify or extend the library code instead of replacing it. This allows me to keep the reliable parts of the code while adding the specific features I need.

As a software integration engineer, it is better to focus on the main integration logic and system communication. Writing basic data structures manually is a waste of time when high-quality libraries are already available and tested.
